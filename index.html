Base framework

The responsive strategy

The goal of Base is to be able to design your layout once, then have that design automatically resize for different device sizes, while retaining clean vertical and horizontal spacing and visual rhythm.

The Base reponsive framework means that you design for a desktop layout, then watch your design automatically resize for smaller screens, tablet and mobile.

Site layouts using Base are best developed using the Chrome browser (though see note on Firefox below).

Copy the entire Base framework to your project folder and modify anything as needed (except perhaps the _mixins.scss file). No CSS overriding should be necessary, you have full style control from the ground up.

Step 1: Set your baseline height

The current baseline height is:

To create visual harmony for your design, a baseline measure is established at the deisgn phase for vertical and horizontal rhythm. It serves as the basis for all spacing and sizing for your design.

The baseline height is set by:

  • Adjusting the $gutterWidth variable in base.scss / desktop.scss / table.scss sup
  • By default the $baseline variable is the same as the $gutterwidth value, to create a linked vertical and horizontal rhythm. This baseline value is applied to the font-size and established the rem size for that device.
  • If necessary, set the $baseline variable to make the baseline/ vertical spacing independent of the columns/ horizontal spacing. This is essntial for mobile, optional for the other devices.

Step 2: Set your typographic spacing and sizing in terms of rems in base.scss

1 rem = 1 baseline unit

You should have worked out with your designer a mockup for the project, worked out your type heirarchy and possibly created the HTML DOM structure of your site before beginning the styling of those elements with the Base .scss files.

Set all your main font-sizes, block sizes and spacing on elements in terms of rems The framework helps you design responsively, as you are no longer thinking in terms of pixel measurements which have to be adjusted from device to device. Use this page to set your typography below.

You can use ems for responsive sizing, though from a design perspective, sizing everything from a clear baseline unit will give you more visually consistent results.

You no longer think in terms of pixels, as the pixels change from device to device. Create you layout once and watch your layout respond to device sizes automatically.

This single baseline value can be altered and adjusted at any point in the development process and automatically affect your whole design.

Step 3: Set up your responsive columns layout in _layout.scss

_layout.scss processes your column widths and offset for each device width. Columns can be declared with whole or 0.5 widths if that is helpful to you. See the examples below.

Note: the _guides.scss file and _test.scss files imported into this file are only for use with this page, and are useful for your initial type configuration. Remove when no longer needed.


// A 4-column-wide <div>, offset by 2 column left margin.

div{
    @include column(2,4);

}

Once you have done this, check your design in Firefox. Gecko browsers renders sub-pixels differently to Webkit and Blink browsers, you may need to slightly adjust your $gutterWidth percentage so the layout works cross-platform.

Step 4: style your layout in base.scss

All styling of elements can be done in this file. You may need to wokr between base.scss and _layout.scss as you layout your page.

Step 5: adjust for devices in .scss stylesheets

Override properties in elements for specific particular devices by applying styles in desktop.scss / tablet.scss / mobile.scss

Using this page

Guides

Simply add <body class="guides"> to your HTML page to view the baseline and column guides overlay. This has limited use as the sub-pixel rendering of browsers means that the baseline is not completely accurate.

Helper elements

This page helps you prepare your type heirarchy and sizing in terms of rems. Once you have set up these styles up, you can remove this page from your project, including the helper elements: _test.scss, demo.js, _guides.scss (though you may wish to keep _guides.scss if you find it helpful for working on your layout.

Typography

Premise

Font-sizes are best set in rems, in order to resize responsively and automatically for all device sizes. (You can still set in pixels or ems if you wish).

As you design may have font-sizes determined in pixels, use this page to determine how that pixel value is set in rems.

.scss files

The typography CSS is broken into separate files in this initial setup – _type.scss, _buttons.scss, _forms.scss, _menus.scss, _fonts.scss, _tooltips.scss – and imported into base.scss. These styles could simply be copied directly into base.scss and reorganised – do whatever is easiest for you.
